HTTPS uses CONNECT. Chrome cannot inject headers on CONNECT — the extension
      answers the proxy 407 with your session ID (same as
      curl -U 'session1234:session'). Open Options
      and set port 8081 for Go (8080 for Node). Reload the
      extension after changes. After browsing, the service worker console should
      show onAuthRequired and authSupplied.
      Unsafe attempt to load URL … chrome-error://chromewebdata/
      is a side effect of ERR_TUNNEL_CONNECTION_FAILED, not the
      root cause — check service worker for networkError hint.
      No 407 usually means TCP/scheme/IP allowlist — not session auth yet.
