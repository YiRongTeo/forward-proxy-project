Forward Proxy Settings

Configure which proxy server the extension routes Chrome through.

Important: Port 8081 is the forward proxy (HTTPS CONNECT). Port 9001 is the admin API only (/health, /sessions/:id). Using 9001 in this extension causes ERR_TUNNEL_CONNECTION_FAILED and Chrome never receives 407.

Hostname only — no http://, no port, no username/password.

HTTPS proxy scheme (when go-proxy tls.certFile is set): the certificate must match your FQDN and be trusted by Chrome (OS trust store). curl may work with --proxy-insecure; Chrome has no equivalent — import your CA or use plain http on an internal network.

407 test (from the same machine as Chrome):
curl -v -x https://HOST:8081 https://example.com -o /dev/null
Do not use -U or --proxy-insecure. You should see HTTP/1.1 407 Proxy Authentication Required after the TLS handshake. If curl cannot connect or TLS fails, Chrome will also fail before any 407. If curl gets 403 instead of 407, your client IP may be blocked in allowedClientIps on the server.
